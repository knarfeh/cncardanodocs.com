Commitments

data Commitment Source #

Commitment is a message generated during the first stage of GodTossing. It contains encrypted shares and proof of secret.

There can be more than one share generated for a single participant.

Shares

type InnerSharesMap = HashMap StakeholderId (NonEmpty (AsBinary DecShare)) Source #

Each node generates several SharedSeeds, breaks every SharedSeed into Shares, and sends those encrypted shares to other nodes (for i-th commitment at i-th element of NonEmpty list). Then those shares are decrypted.

type SharesMap = HashMap StakeholderId InnerSharesMap Source #

In a SharesMap, for each node we collect shares which said node has received and decrypted:

  • Outer key = who decrypted the share
  • Inner key = who created the share

Let's say that there are participants {A, B, C}. If A has generated a secret and shared it, A's shares will be denoted as Ab and Ac (sent correspondingly to B and C). Then node B will decrypt the share and get Ab_dec; same for other nodes and participants. In the end, after the second phase of the protocol completes and we gather everyone's shares, we'll get the following map:

{ A: {B: Ba_dec, C: Ca_dec}
, B: {A: Ab_dec, C: Cb_dec}
, C: {A: Ac_dec, B: Bc_dec}
}

(Here there's only one share per node, but in reality there'll be more.)

type SharesDistribution = HashMap StakeholderId Word16 Source #

This maps shareholders to amount of shares she should issue. Depends on the stake distribution.
