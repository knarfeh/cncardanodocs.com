-> (HasConfiguration => m r) 
-> m r 

Come up with a HasConfiguration constraint using a Configuration. The Configuration record can be parsed from JSON or Yaml, and used to get a GenesisSpec, either from the file itself or from another file: the canonical JSON encoding of a mainnet genesis.

If the canonical JSON source is given, then it will be hashed and checked against expected hash (which is also part of configuration in this case).

If the configuration gives a testnet genesis spec, then a start time must be provided, probably sourced from command line arguments.
