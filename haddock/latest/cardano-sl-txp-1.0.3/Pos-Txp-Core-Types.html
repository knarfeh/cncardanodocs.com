data TxPayload Source #

Payload of Txp component which is part of main block. Constructor is unsafe, because it lets one create invalid payload, for example with different number of transactions and witnesses.

Constructors

UnsafeTxPayload 

Fields

  • _txpTxs :: !(MerkleTree Tx)

    Transactions are the main payload.

  • _txpWitnesses :: ![TxWitness]

    Witnesses for each transaction. The length of this field is checked during deserialisation; we can't put witnesses into the same Merkle tree with transactions, as the whole point of SegWit is to separate transactions and witnesses.

    TODO: should they be put into a separate Merkle tree or left as a list?
